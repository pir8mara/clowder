# Contributing

Clowder accepts contributions under AGPL-3.0-only using the Developer Certificate of Origin.

## License

By contributing to Clowder, you agree that your contribution is submitted under the project's AGPL-3.0-only license.

## Developer Certificate of Origin

All commits must include a `Signed-off-by` line certifying the Developer Certificate of Origin 1.1.

Use:

```bash
git commit -s
```

or manually add:

```text
Signed-off-by: Your Name <you@example.com>
```

## No CLA

Clowder does not require a Contributor License Agreement.

Clowder does not require copyright assignment. Contributors retain copyright to their contributions while licensing them under AGPL-3.0-only.

## Dependency policy

Core dependencies should be compatible with AGPL-3.0-only and should not encumber redistribution, auditing, local operation, or community modification.

Avoid adding core dependencies that are proprietary, source-available but not open source, non-commercial only, or otherwise incompatible with AGPL-3.0-only.

## Project posture

Clowder is for authorized security assessment by knowledgeable operators. Contributions should preserve the project's safety model, evidence-first design, local-first posture, and scope-constrained workflow.
